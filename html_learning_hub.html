What is HTML?

HTML (HyperText Markup Language) is the standard markup language for documents designed to be displayed in a web browser.

HTML describes the structure of a web page semantically and originally included cues for the appearance of the document.

With HTML, you can create structured documents by denoting structural semantics for text such as headings, paragraphs, lists, links, quotes, and other items.

index.html 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First HTML Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is my first HTML page.</p>
</body>
</html>

Document Type Declaration

All HTML documents must start with a document type declaration: <!DOCTYPE html>. This helps browsers understand what type of document they're dealing with.

The HTML Element

The <html> element is the root element of an HTML page and contains all other HTML elements (except for the DOCTYPE).

The Head Section

The <head> element contains meta information about the HTML document that isn't displayed on the page, such as the title, character encoding, styles, scripts, and more.

The Body Section

The <body> element contains all the content that is shown to users, such as headings, paragraphs, images, hyperlinks, tables, lists, etc.

Semantic Structure

Modern HTML5 includes semantic elements like <header>, <nav>, <main>, and <footer> that give meaning to different parts of the page.

Proper HTML structure improves accessibility, SEO, and maintainability of your website.

HTML Forms

Form Elements

HTML forms are used to collect user input. The <form> element is a container for different types of input elements.

  • <input> Text, password, checkbox, radio, etc.
  • <textarea> Multi-line text input
  • <select> Dropdown list
  • <button> Clickable button
  • <label> Label for input elements

Form Attributes

Important form attributes include:

  • action: Where to send the form data
  • method: HTTP method (GET or POST)
  • enctype: How form data is encoded

Semantic HTML

Why Use Semantic HTML?

Semantic HTML uses tags that convey meaning about the structure and content of your web page, not just how it looks.

Benefits of Semantic HTML:

  • Improves accessibility for users with disabilities
  • Enhances SEO (Search Engine Optimization)
  • Makes code easier to understand and maintain
  • Provides consistent structure across web pages
  • Helps with responsive design

Non-Semantic vs. Semantic

Non-Semantic Approach 
<div class="header">
    <div class="navigation">...</div>
</div>

<div class="main-content">
    <div class="article">...</div>
    <div class="sidebar">...</div>
</div>

<div class="footer">...</div>
Semantic Approach 
<header>
    <nav>...</nav>
</header>

<main>
    <article>...</article>
    <aside>...</aside>
</main>

<footer>...</footer>

Semantic HTML5 Page Structure

<header>
Page header with logo, title, navigation
<nav>
Navigation links
<main>
<section>
Grouped content
<article>
Independent, self-contained content
<aside>
Sidebar, related content
<footer>
Page footer with copyright, links, etc

Semantic HTML Best Practices:

Always use the most appropriate semantic element for the content's purpose, not its appearance. This improves accessibility and SEO.
